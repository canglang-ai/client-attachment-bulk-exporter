# 批量导出附件（MokaHR 候选人 PDF）浏览器插件

## 功能

- 在插件弹窗里导入 **Excel(.xlsx)** / **CSV**，读取申请 ID（例如 `588932532`）
- 按 URL 模板逐个打开详情页
- 自动点击“附件信息”里 PDF 的下载按钮
- 自动把下载文件统一保存到同一子文件夹（默认 `mokahr-pdfs/`），并按申请 ID 命名

## 安装（Chrome / Edge）

1. 打开 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本目录（`批量导出附件/extension`）

## 使用

1. 登录 `app.mokahr.com`
2. 点插件图标打开弹窗
3. 选择导入文件（CSV 或 XLSX），确保里面包含申请 ID 列
4. 如有需要，修改 URL 模板（默认已填好）
5. 点击 Start

## 下载保存位置

- 插件会把 PDF 文件名建议为 `mokahr-pdfs/<申请ID>-<name>.pdf`（子文件夹可在弹窗里改）
- 实际保存位置仍然在浏览器“默认下载目录”下（这是浏览器限制）

## 表格格式建议

- 第一列是申请 ID：`application_id`
- 可选第二列是候选人姓名：`name`
- 可选第三列是应聘职位名称：`job_title`（或 `title/position/职位/岗位/应聘职位名称` 等表头也支持）

示例 CSV：

```csv
application_id,name,job_title
588932532,班勇锐,设计师
588932533,张三,测试工程师
```

无表头时，默认按列顺序解析为：

- 第 1 列：申请ID
- 第 2 列：应聘职位名称
- 第 3 列：姓名

## 常见问题

- **点了 Start 没反应/提示找不到下载按钮**：通常是未登录、没有 PDF 附件，或页面结构更新导致按钮 class 变化。此时把按钮在 DevTools 里看到的 class/DOM 发我，我会把 `extension/content.js` 的选择器改得更精准。
- **弹出“保存”对话框但没有自动点保存**：这是浏览器/系统原生对话框，插件无法自动点击。请到浏览器设置关闭“下载前询问每个文件的保存位置”（Chrome: 设置 -> 下载内容）。
- **下载被浏览器拦截**：确认站点允许多文件下载，并关闭“下载前询问保存位置”，再重试。
